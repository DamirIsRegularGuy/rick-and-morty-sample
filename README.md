# Rick and Morty — улучшённая версия

## Что изменилось

**Архитектура**
- Callback-based Retrofit → suspend-функции + корутины.
- Добавлен `MainViewModel` (StateFlow) вместо логики прямо в Activity.
- `CharacterRepository` — тонкий слой над API.
- Пакеты разложены: `model/`, `network/`, `ui/main/`, `ui/detail/`, `adapter/`.

**Функционал**
- **Пагинация**: подгрузка следующих страниц при скролле до конца списка.
- **Поиск по имени** через `SearchView` в тулбаре (debounce 400ms, без лишних запросов на каждую букву).
- **Экран деталей персонажа** (`DetailActivity`) по клику на карточку: пол, тип, происхождение, последнее известное местоположение, количество эпизодов (с правильным русским склонением), дата добавления в базу (отформатированная).
- **Pull-to-refresh** (SwipeRefreshLayout).
- Состояния загрузки / ошибки / "ничего не найдено" вместо просто Log.d в консоль.
- Логирование HTTP-запросов через `HttpLoggingInterceptor` — удобно для отладки.

**Исправлен баг**
- В оригинальном адаптере `origin` и `location` были перепутаны местами (label "Last known location" показывал `origin.name`, а должен — `location.name`). Исправлено.

**Дизайн**
- `DiffUtil`/`ListAdapter` вместо ручного `notifyDataSetChanged`.
- Экран деталей: hero-фото с градиентом и именем поверх, карточка с рядами "лейбл + значение" в стиле остального приложения.

**Новое: избранное**
- Сердечко на карточке в списке и в меню экрана деталей.
- `FavoritesRepository` (`data/FavoritesRepository.kt`) — синглтон-хранилище на `SharedPreferences` + Gson,
  хранит персонажей целиком (не только id), поэтому экран "Избранное" открывается без единого запроса к сети.
  Инициализируется один раз в `RickAndMortyApp.onCreate()`, поэтому список, детали и избранное всегда синхронны.
- Новый экран `FavoritesActivity` — открывается по иконке в тулбаре главного экрана.

**Новое: фильтр по статусу**
- Чипы "Все / Живые / Мертвые / Неизвестно" под тулбаром на главном экране.
- Использует тот же query-параметр API (`status`), что и поиск по имени; фильтр и поиск можно комбинировать.

**Новое: шеринг персонажа**
- Кнопка "Поделиться" на экране деталей — стандартный `ACTION_SEND` с текстом о персонаже (имя, статус, вид, локация).

## Что можно добавить дальше (если понадобится)
- Room для офлайн-кеша списка персонажей (сейчас офлайн работает только для избранного).
- Фильтр по виду (species) — работает по тому же принципу, что и status.
- Свайп для удаления из избранного на экране "Избранное".
